# Дополнительные исправления PlotApp

Дата: 2026-05-01

## Что изменено

### 1. Экспорт PNG/SVG при высоком DPI

Исправлена причина дефекта, показанного на приложенном снимке экспорта при 640 dpi.

Раньше `QImage` получал DPI до начала отрисовки. Qt пересчитывал point-size шрифты через DPI устройства, поэтому подписи осей, деления и легенда становились непропорционально большими, а геометрия рабочей области оставалась в пикселях.

Теперь экспорт работает так:

- сначала график рисуется в логическом размере текущей видимой рабочей области;
- затем готовый снимок масштабируется до целевого размера экспорта;
- DPI записывается в PNG только после отрисовки как metadata.

Это сохраняет те же визуальные пропорции, что и в предпросмотре и на экране.

Изменённые файлы:

- `src/ui/PlotCanvasWidget.cpp`
- `src/ui/PlotCanvasWidget.h`

### 2. A4-режим рабочей области для скриншотов

Добавлены команды для подстройки центральной рабочей области под соотношение сторон A4:

- `View -> Fit canvas to A4 landscape`
- `View -> Fit canvas to A4 portrait`
- `View -> Release canvas aspect lock`

Также добавлена панель `Canvas` с кнопками:

- `A4 screenshot`
- `A4 portrait`
- `Free canvas`

Режим не открывает закрытые панели/терминал. Если консольная панель закрыта, рабочая область занимает максимум доступного места. Если панели открыты, рабочая область вписывается в оставшееся пространство.

Изменённые файлы:

- `src/ui/MainWindow.cpp`
- `src/ui/MainWindow.h`

### 3. Редактирование легенды

Исправлено диалоговое окно свойств слоя: многострочное поле легенды теперь автоматически увеличивается по высоте при добавлении строк через Enter.

Также добавлено быстрое редактирование легенды прямо с графика: одиночный клик по окну легенды открывает безопасный многострочный редактор. Перетаскивание легенды сохранено: если мышь была сдвинута, считается перемещение, а не редактирование.

Изменённые файлы:

- `src/ui/LayerPropertiesDialog.cpp`
- `src/ui/LayerPropertiesDialog.h`
- `src/ui/PlotCanvasWidget.cpp`
- `src/ui/PlotCanvasWidget.h`
- `src/ui/MainWindow.cpp`
- `src/ui/MainWindow.h`

### 4. Автосохранение в cache

Добавлено автосохранение проекта в cache:

- файл: `autosave.plotapp` внутри `QStandardPaths::CacheLocation`;
- интервал: 15 секунд;
- дополнительное сохранение при закрытии окна и после ключевых операций изменения проекта;
- при следующем запуске окно пытается автоматически восстановить этот cache-проект.

Пустой новый проект без пользовательского содержимого не перезаписывает autosave.

Изменённые файлы:

- `src/ui/MainWindow.cpp`
- `src/ui/MainWindow.h`

### 5. Новый проект в новом окне

Пункт меню `New project` заменён на `New project in new window`.

Теперь создание нового проекта открывает отдельное окно с чистым проектом, а старое окно остаётся открытым.

Изменённые файлы:

- `src/ui/MainWindow.cpp`
- `src/ui/MainWindow.h`

### 6. Белые названия осей и заголовка в тёмной теме

Усилена отрисовка rich text для заголовка, подписей осей и легенды: цвет теперь явно задаётся внутри HTML-блока, а не только через stylesheet `QTextDocument`.

Это должно устранить случаи, когда текст на тёмной теме отображался не тем цветом.

Изменённые файлы:

- `src/ui/PlotCanvasWidget.cpp`

### 7. Тесты и проверки

Добавлен статический регрессионный тест GUI-контрактов:

- `tests/gui_static_checks.sh`

Он подключён в `CMakeLists.txt` как отдельный тест `plotapp_gui_static_checks`, если проект собирается через CMake.

## Проверки

В текущей среде выполнены:

- ручная сборка core-тестов без CMake;
- ручная сборка plugin `.so` модулей;
- запуск существующего набора `plotapp_tests`;
- запуск `tests/gui_static_checks.sh`.

Результат:

```text
All tests passed.
GUI static checks passed.
```

## Ограничения проверки

Полный запуск Qt GUI-приложения и реальный экспорт через окно программы не были выполнены, потому что в среде нет Qt6 Widgets development headers/libraries:

```text
Qt6Widgets pkg-config not found
no_qt_headers
```

Поэтому GUI-часть проверена статически и через анализ изменённых исходников. Core-логика проекта и существующие регрессионные тесты проходят.
